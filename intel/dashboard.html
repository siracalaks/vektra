Kurulum rehberi — 4 adım
Bu projeyi sıfırdan kuracaklar için. Sırayla yap, "ne yapacağım?" kalmaz.
  1. Supabase projesi aç — supabase.com (ücretsiz). Yeni proje oluştur, bölge olarak yakınını seç.
  2. Tabloları oluştur — Supabase panelinde sol menüden SQL Editor → + New query → aşağıdaki SQL'i 📋 Kopyala ile al, kutuya yapıştır → sağ altta Run (veya Ctrl+Enter). Yeşil “Success” görürsen tamamdır. Hepsi tek seferde çalışır — parça parça değil.
    Bu tek SQL şu 4 şeyi sırayla oluşturur:
    • vector eklentisi — embedding (anlamsal arama) için
    • Tablo 1: intel_items — motorun her gün topladığı bulgular (puan, özet, para skoru…)
    • Tablo 2: library_embeddings — kalıcı kütüphane kartları (oto-büyüyen katalog buraya yazılır)
    • Fonksiyon: match_all — “Sor” için iki tabloda anlamsal arama · + güvenlik kuralları (RLS)
  3. Anahtarları bağla — Supabase → Settings → API → Project URL + anon public anahtarını kopyala → bu panelde Ayarlar sekmesine yapıştır → Kaydet. (Dashboard'u bağlar.)
  4. Motoru kur (n8n) — n8n'e 00-meta/engine-node.js kodlu bir Code node'lu workflow ekle, CONFIG'e anahtarları gir (Supabase service_role, Anthropic, Gemini, opsiyonel kaynaklar), günlük zamanla + Active yap. Aynı şekilde 00-meta/ask-node.js ile "Sor" webhook'unu kur.
Bitince: İstihbarat motorun bulgularıyla, Sor doğal dil aramayla, Kütüphane oto-büyüyen katalogla canlı olur. Detay: repo 00-meta/engine-setup-guide.md.

7 Kaynak — nereyi tarar
GitHub (trend repolar), Hacker News (öne çıkanlar), YouTube (Data API), Product Hunt (yeni ürünler), Reddit ve Instagram (Apify üzerinden — OAuth derdi yok). Hepsi geliştirici/AI dünyasının nabzı. Kaynak başına döngüsel (round-robin) seçim, üst sınır 40 → tek bir kaynak akışı domine etmez.
Motor (n8n, her sabah)
Bir n8n iş akışı günde bir kez çalışır: harvest → derin oku → beyin → embed → Supabase'e yaz → WhatsApp özeti. Sonunda {harvested, stored, embedded, notified} döner. Senin makinen kapalıyken bile bulutta çalışır.
Derin okuma
Yüzeysel başlığa göre puan vermez. GitHub için README'yi, web için Firecrawl ile sayfayı gerçekten okur; YouTube/Instagram zaten kendi metnini taşır. Beyin böylece içeriğin özünü değerlendirir.
Beyin — Claude Haiku
Her bulguyu okur ve üretir: ilgi puanı (0-100), kısa özet, “neden önemli”, etiketler, ve para radarı — fırsat türü (tool / idea / lead / trend / clone) + para skoru. Hızlı ve ucuz model (claude-haiku-4-5).
Embed — Gemini 768
Her satır kaydedilirken gemini-embedding-001 ile 768 boyutlu bir vektöre çevrilir. Bu sayede “Sor” kelime değil anlam üzerinden eşleştirir (pgvector).
Supabase — hafıza
Bulut veritabanı. intel_items = günlük istihbarat akışı (özet, puan, para skoru, embedding…). library_embeddings = kalıcı kütüphane kartları. match_all RPC iki tabloda birden anlamsal arama yapar. Güvenlik (RLS): dışarıdan sadece okunur + saved güncellenir; motor gizli anahtarla yazar.
📚 Kütüphane sekmesi
Çevrimdışı 194 elle seçilmiş kart + buluttan otomatik büyüyen kartlar (🤖 oto rozeti). Aşama / tür / alan filtreleri. İnternetsiz bile temel katalog açılır.
🛰️ İstihbarat sekmesi
Supabase'den canlı akış. Kaynak / fırsat / ilgi / para / YENİ / Bugün filtreleri. Bir bulguyu beğenince “→ kütüphaneye ekle” ile işaretlersin (saved=true).
💬 Sor
Doğal dille soru → Gemini embed → pgvector ile kütüphane+istihbarat taranır → Haiku kaynaklı Türkçe cevap üretir. Güvenli: kaynaklar yalnızca veri olarak okunur, içlerindeki komutlar ASLA çalıştırılmaz.
📱 WhatsApp sabah özeti
Motor her çalıştığında en yüksek ilgili 5 bulguyu Evolution API ile telefonuna yollar; “📚 N araç kütüphaneye eklendi” bilgisini de iletir. Dashboard'u açmadan günü yakalarsın.
🔄 GitHub sync (opsiyonel, günlük)
Ayrı bir n8n akışı Supabase kütüphanesini repoya yansıtır (.md kartlar + registry). Veriyi sürüm kontrolünde yedekler — bulut tek nokta hata olmaz.
🔌 Vektra MCP — her projede
vektra_ask aracı (saf-Python stdio MCP, kullanıcı kapsamında global kayıtlı) güvenli Sor webhook'unu çağırır. Kütüphaneyi her projeye kopyalamazsın; merkezde tutar, ihtiyaç anında sorarsın. Claude cevabı okur ve senin onayınla ilgili kısmı projeye çeker.
🤖 Oto-kütüphaneci — kendi kendine büyüme
Motor, nitelikli bulguları doğrudan kütüphaneye ekler. Kapı (gate): ilgi ≥ 80 + güvenli tür (library / app / mcp / reference / skill) + geçerli aşama + embedding var + zaten ekli değil. Güvenli: yalnızca beynin sınırlı özeti saklanır (ham içerik / gizli komut değil); kurulum tarifi asla saklanmaz/çalıştırılmaz — kataloglamak ≠ kurmak. Her oto-kart watch durumunda, skiller “🔒 kurmadan incele” rozetli.

🧭 Bunu nasıl kullanırım

Günlük pratik akış — cihaza hâkim olmanın yolu.

  1. Sabah: WhatsApp özetine bak (veya İstihbarat sekmesi → “Bugün” filtresi). Gece ne çıktı, 30 saniyede gör.
  2. Beğendiğini işaretle: ilginç bir bulguda “→ kütüphaneye ekle”. Sonra terminalde python 00-meta/inbox-sync.py → taslak kart 99-inbox/'a düşer, incele & kalıcılaştır.
  3. Bir işe başlarken Sor'a sor: örn. “görsel→video için hangi API?” / “Supabase'de auth nasıl?”. Kütüphane + istihbarattan sentezlenmiş, kaynaklı cevap gelir.
  4. Başka projelerde: o projede Claude'a “vektra'ya sor: …” de → MCP merkezî beyni sorgular, en uygun aracı/kaynağı getirir, onayınla projene çeker. Kütüphaneyi kopyalamana gerek yok.
  5. Hiçbir şey yapmasan da: kütüphane kendi kendine büyür (ilgi ≥ 80 + güvenli tür + geçerli aşama → otomatik kart, 🤖 rozetli). Sen sadece kullanırsın.

📊 Canlı durum

Supabase'e bağlıysan canlı sayılar burada görünür. Ayarlar sekmesinden URL + anon anahtarını gir.

🔐 Güvenlik & maliyet

  • Anahtarlar nerede: n8n kimlik bilgilerinde / repo dışı .env'de — git'e ASLA girmez.
  • Beyin & embed: Claude Haiku (puanlama) + Gemini (embedding). Çoğu kaynak ücretsiz/self-host.
  • Maliyet: ~$3-8 / ay (günde birkaç sent). Supabase + n8n + Apify free/ucuz katmanlarında.
  • Erişim: siteyi Basic Auth arkasına alabilirsin; “Sor” webhook'u token korumalı.
  • Veri güvenliği (RLS): dışarıdan anon yalnızca okur + saved günceller; yazma sadece motorun gizli service anahtarıyla.
